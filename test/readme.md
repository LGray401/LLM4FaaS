# get_function_output.py
1. This script runs the generated functions located in a specified directory and store the console output in log files.
2. Specifically, the console output is stored into three parts: stdout, stderr, and the return code.
3. We set a 30 seconds timeout for each function.



# test_result.py
1. This script analyzed the log files generated by get_function_output.py and generate a summary of the test results into csv.
2. It checks if all standard outputs are contained in the actual function outputs. 
   1. The standard output includes the essential information from the smart device based on our project setup.
   2. The actual outputs can have some additional information like personalized context statements.
3. Specifically, it requires the TASK_NAME and the directories of both the generated log files and the standard log files.
4. It records the following information of log file: 
   1. The model name that generates the function
   2. The task name
   3. The index of user
   4. The error status of the function
   5. The success rate of the function compared to the standard answer according to user responses.
   6. The timestamp of function generation
   7. the path of log file.
